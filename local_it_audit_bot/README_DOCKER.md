# Docker-версия бота для ИТ-аудита

Этот проект упакован в Docker для удобного развертывания. Он состоит из двух сервисов:
1. **Ollama** — локальный сервер с ИИ-моделью (LLaVA)
2. **Bot** — сам телеграм-бот

## Требования
- Docker и Docker Compose
- Для работы с изображениями желательна видеокарта NVIDIA (опционально)
- Токен телеграм-бота от @BotFather

## Быстрый старт

### 1. Подготовка токена
Создайте файл `.env` в папке проекта:
```bash
echo "TELEGRAM_BOT_TOKEN=ваш_токен_здесь" > .env
```

### 2. Запуск
```bash
docker-compose up -d
```

Первый запуск может занять время, так как будет скачан образ Ollama и модель LLaVA (~4-5 ГБ).

### 3. Проверка статуса
```bash
docker-compose logs -f
```

## Конфигурация

### Для видеокарт NVIDIA
В `docker-compose.yml` уже раскомментирован блок для GPU. Убедитесь, что у вас установлен [NVIDIA Container Toolkit](https://docs.nvidia.com/datacenter/cloud-native/container-toolkit/install-guide.html).

### Только CPU (без видеокарты)
Если у вас нет GPU или вы на macOS:
1. Откройте `docker-compose.yml`
2. Закомментируйте блок `deploy: ...` внутри сервиса `ollama`
3. Раскомментируйте строку `# command: ["serve"]`

## Как это работает
- Бот отправляет фото и текст в локальный сервис Ollama через внутреннюю сеть Docker.
- Модель LLaVA анализирует изображение и формирует отчет.
- Все данные остаются внутри вашего компьютера/сервера.

## Остановка и удаление
```bash
# Остановить
docker-compose down

# Остановить и удалить данные модели (освободить место)
docker-compose down -v
```

## Доступ из интернета (без статического IP)
Так как бот работает внутри Docker-сети, наружу он не выступает. Для доступа извне используйте туннели:

### Вариант А: Cloudflare Tunnel (рекомендуется)
Установите `cloudflared` на хост-машину и пробросьте порт бота (если бы он был веб-сервисом), но так как это телеграм-бот, ему нужен только исходящий доступ в интернет, который уже есть по умолчанию. **Никаких дополнительных настроек сети не требуется!** Телеграм сам постучится к вашему боту.

### Вариант Б: Tailscale / ZeroTier
Если нужно управлять сервером удаленно, установите Tailscale на хост-машину.

## Обновление модели
По умолчанию используется модель `llava`. Если хотите сменить её:
1. Измените переменную `MODEL_NAME` в `docker-compose.yml`.
2. Перезапустите контейнер: `docker-compose restart bot`.
3. Внутри контейнера ollama выполните `ollama pull <новая_модель>` (если она еще не скачана).
